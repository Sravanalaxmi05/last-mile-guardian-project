# Last-Mile Guardian

A Gemma 4-powered alert-to-action layer for floods. Converts official flood alerts into personalized survival action cards for vulnerable people.

## Run & Operate

- `pnpm --filter @workspace/api-server run dev` — run the API server (port 8080)
- `pnpm --filter @workspace/last-mile-guardian run dev` — run the React frontend (port 24730)
- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from the OpenAPI spec

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- API: Express 5
- Frontend: React + Vite + TailwindCSS + shadcn/ui
- AI: Google Generative AI (@google/generative-ai) — Gemma 3 27B
- Validation: Zod, drizzle-zod
- API codegen: Orval (from OpenAPI spec)
- Build: esbuild (CJS bundle)

## Where things live

- `lib/api-spec/openapi.yaml` — single source of truth for API contracts
- `artifacts/api-server/src/routes/` — Express route handlers
- `artifacts/api-server/src/data/personas.ts` — the 3 vulnerability personas (Asha, Imran, Meena)
- `artifacts/api-server/src/lib/demoCards.ts` — high-quality deterministic demo outputs
- `artifacts/api-server/src/lib/gemmaClient.ts` — Google Generative AI client
- `artifacts/last-mile-guardian/src/` — React frontend

## Architecture decisions

- No database — app is stateless (all data derived from Gemma calls or demo mode)
- Demo mode always works without any API key (uses pre-written high-quality outputs)
- Gemma mode activates automatically when GOOGLE_API_KEY or GEMMA_API_KEY is set
- Falls back to demo mode if Gemma fails
- Zod validation at route level for request bodies

## Product

Last-Mile Guardian converts a single official flood alert into personalized survival action cards for three vulnerable personas: Asha (68, diabetic, ground floor, alone), Imran (wheelchair user, first floor), and Meena (mother with two children, water outside). Output includes SMS card, IVR voice script, WhatsApp family card, volunteer rescue handoff, and offline checklist.

## User preferences

- Keep it simple and reliable — no maps, no login, no database, no real SMS/dispatch
- Must work in Demo Mode without any API key
- Non-technical user — app must be self-explanatory

## Gotchas

- Set GOOGLE_API_KEY or GEMMA_API_KEY to enable real Gemma 4 mode
- Without API key, the app runs in Demo Mode with pre-written high-quality outputs
- The frontend uses hooks from @workspace/api-client-react (generated by Orval)
- Run codegen after any OpenAPI spec changes: `pnpm --filter @workspace/api-spec run codegen`

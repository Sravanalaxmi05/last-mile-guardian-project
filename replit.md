# Last-Mile Guardian

A Gemma 4 alert-to-action layer for floods. Converts official flood alerts into personalized survival action cards for vulnerable people.

## Run & Operate

- `pnpm --filter @workspace/api-server run dev` — run the API server (port 8080)
- `pnpm --filter @workspace/last-mile-guardian run dev` — run the React frontend (port 24730)
- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from the OpenAPI spec
- `pnpm eval:gemma4` — evaluate the live API against alert fixtures

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- API: Express 5
- Frontend: React + Vite + TailwindCSS + shadcn/ui
- AI: Gemma 4 via provider adapter.
  - `GEMMA_PROVIDER=google-ai` for Google AI SDK
  - `GEMMA_PROVIDER=openai-compatible` for a Gemma 4 endpoint served by vLLM/Hugging Face/other compatible hosts
  - `GEMMA_MODEL_ID` must be an actual Gemma 4 model ID
  - server refuses live mode for non-Gemma-4 model IDs
- Validation: Zod, deterministic life-safety validator, drizzle-zod
- API codegen: Orval (from OpenAPI spec)
- Build: esbuild (CJS bundle)

## Where things live

- `lib/api-spec/openapi.yaml` — single source of truth for API contracts
- `artifacts/api-server/src/routes/` — Express route handlers
- `artifacts/api-server/src/data/personas.ts` — the 3 vulnerability personas (Asha, Imran, Meena)
- `artifacts/api-server/src/lib/demoCards.ts` — deterministic demo outputs
- `artifacts/api-server/src/lib/gemma/` — provider adapters, prompts, pipeline, schemas, metadata, and safety validation
- `artifacts/last-mile-guardian/src/` — React frontend
- `eval/` — fixtures and live evaluation runner

## Architecture decisions

- No database — app is stateless (all data derived from live calls or deterministic demo mode)
- Live generation is staged: alert extraction, decision packet, cards, schema validation, safety validation
- Demo mode always works without any API key and is visibly labeled
- If live Gemma 4 fails, the app visibly falls back to deterministic safe cards and shows the fallback reason
- `ENABLE_DEMO_MODE=false` disables fallback for strict live demos
- Zod validation exists at request boundaries and model-artifact boundaries

## Product

Last-Mile Guardian converts a single official flood alert into personalized survival action cards for three vulnerable personas: Asha (68, diabetic, ground floor, alone), Imran (wheelchair user, first floor), and Meena (mother with two children, water outside). Output includes SMS card, IVR voice script, WhatsApp family card, volunteer rescue handoff, and offline checklist.

## User preferences

- Keep it simple and reliable — no maps, no login, no database, no real SMS/dispatch
- Must work in Demo Mode without any API key
- Non-technical user — app must be self-explanatory

## Gotchas

- Set a valid Gemma 4 model ID and API key to enable live mode
- Without a valid live configuration, the app runs in visibly labeled Demo Mode
- The frontend uses hooks from @workspace/api-client-react (generated by Orval)
- Run codegen after any OpenAPI spec changes: `pnpm --filter @workspace/api-spec run codegen`
